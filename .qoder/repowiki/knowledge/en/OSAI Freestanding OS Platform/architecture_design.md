- **Boot Contract**: The UEFI bootloader (`boot/uefi`) validates and loads the kernel ELF, passing a standardized `osai_boot_info_t` structure containing the memory map and UART base, which serves as the initial handshake between firmware and kernel.
- **Syscall ABI Enforcement**: A frozen JSON contract (`contracts/qemu-rc-v1.json`) defines the canonical syscall numbers and capability bits. This contract is programmatically validated against both the kernel header (`kernel/include/osai/syscall.h`) and the userspace library (`userspace/lib/osai_user.c`) via Python-based gate scripts to ensure ABI consistency.
- **Initramfs Integration**: The build system uses a shared Python script (`scripts/create-initfs.py`) to generate a read-only FAT-based initramfs that embeds the kernel's expected service descriptors and CPU-AI models, ensuring the kernel's `initramfs` module and userspace `/init` process share a consistent filesystem layout.
- **Unified Build & Test Pipeline**: A top-level `Makefile` orchestrates the compilation of all components (UEFI, Kernel, Userspace) into a single bootable disk image, which is then verified by a suite of QEMU-based integration tests that check telemetry, persistence, and security policies defined in the central contract.
