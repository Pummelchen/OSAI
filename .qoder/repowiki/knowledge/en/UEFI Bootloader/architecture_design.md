- Single entry point `efi_main` in `loader_main.c` orchestrates the full boot sequence: open EFI volume, read `\EFI\OSAI\kernel.elf`, validate ELF64 headers, allocate physical pages for loadable segments, capture the EFI memory map, call `ExitBootServices`, and jump to the kernel entry point.
- `include/uefi_min.h` provides a minimal, self-contained UEFI type and protocol definition layer (system table, boot services, file system protocol, loaded image protocol) without depending on external UEFI SDKs — all types are forward-declared structs with function-pointer members.
- `boot_info.h` is a thin wrapper that re-exports `osai_boot_info_t` from `../../kernel/include/osai/boot_info.h`, establishing a shared ABI contract between the bootloader and the kernel for passing the memory map, kernel physical bounds, and UART base.
- Dependency direction flows inward: `loader_main.c` depends on both `uefi_min.h` (UEFI abstraction) and `boot_info.h` (kernel ABI); no other modules depend on this leaf.
